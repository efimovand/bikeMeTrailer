# BikeMe — промо-трейлер (Remotion)

## Что это
Премиальный промо-**трейлер в стиле презентаций Apple** (вайб Apple Keynote / Ferrari «Luce»):
почти-чёрный фон, много воздуха, медленный уверенный темп, плавные пружинные анимации,
тонкая типографика, глубина (параллакс, мягкие тени, лёгкий блюр), один акцентный цвет.

Это **видео, собранное кодом** через [Remotion](https://remotion.dev) (React/TypeScript → MP4).
Рендерится покадрово, без таймлайн-редактора.

## Что рекламируем
**BikeMe** — Telegram-бот (отдельный проект, лежит в `../bikeMeBot`), который по 3 фото
пользователя с помощью ИИ генерирует фотореалистичный кадр: человек на выбранном мотоцикле
в выбранной экипировке. Трейлер показывает этот путь: **загрузка фото → выбор байка →
сбор экипа → выбор цвета → результат → галерея → телефон → лого магазина**.

Финал — прямой оффер для холодных продаж мотомагазинам: анимация ребрендинга (TG-шапка
бота → «Ваш магазин»), цены (SaaS / Buy-out) и контакт. (Персонализация через проп
`shopName` была удалена — финал общий для всех.)

## Стек и команды
- Remotion `4.0.471`, TypeScript, 9:16 вертикаль **1080×1920**, **30 fps**.
- `@remotion/transitions`, `@remotion/google-fonts` (шрифт **Inter** — замена SF Pro).
- Превью (главный инструмент итераций): `npm run dev` → Remotion Studio, скрабинг ←/→ покадрово.
- Стилл для проверки кадра: `npx remotion still Trailer out/test.png --frame=N`
- Финальный рендер: `npx remotion render Trailer out/trailer.mp4`

## ⚠️ Дизайн-правила (НЕ нарушать — это и есть «эппловость»)
1. Фон почти-чёрный **`#0F1923`** (rgb 15,25,35), много пустого пространства.
2. Медленный уверенный темп; резкое движение только там, где задумано (флик/прокрут лент).
3. Изинг **только `Easing.out` / `Easing.inOut` / `spring`** — **никогда не линейный**
   (исключение: внутри SceneResultsFan `flash` использует linear — намеренно, для «вспышки»).
4. Шрифт Inter, тонкий (300) / полужирный (600-700), крупный, минимум слов.
5. Глубина: параллакс, мягкие тени, лёгкий блюр заднего слоя на переходах.
6. Один акцентный цвет — **оранжевый `#E8660A`** (rgb 232,102,10), точечно. Световой блик по лого.

## Композиция и тайминг (КРИТИЧНО)
- `src/Root.tsx` — регистрирует композицию `Trailer`, задаёт `durationInFrames=913`, fps, размер.
- `src/Trailer.tsx` — `TransitionSeries`: сцены + переходы. Основной переход `zoomThrough` (custom) + `springTiming({damping:200, durationInFrames:24})`.
  - **Исключение:** переход SceneResultsFan → Scene4Deck в данный момент закомментирован (обычный zoomThrough). При необходимости плавного стыка карточки — использовать `fade()` с `linearTiming`.
- **Переходы «съедают» кадры**: общая длина = Σ(длительности сцен) − Σ(длительности переходов).
  При изменении длительности любой сцены или перехода — **пересчитать и обновить `durationInFrames` в `Root.tsx`**, иначе вылезет чёрный хвост или обрежется финал.
- Все тайминги — в кадрах при 30 fps. Анимации: `interpolate(frame, [from,to], [a,b], {easing})` или `spring()`.

## Сцены (`src/scenes/`)
1. **Scene1Intro** — старт: TG-строка ввода со скрепкой → курсор кликает → пружиной появляется фото лица (`generations/20.png`), в строке «✓ Фото загружено». Заголовок «Начни с фото.».
2. **SceneBikePick** — выбор байка. Две CS-кейс-ленты:
   - верхняя — бренды (`public/brands.png` прокручивается влево, садится на **BMW**);
   - нижняя — силуэты байков (оранжевые PNG, прокрутка влево, садится на **S1000RR**);
   - подсветка выбранного (`#D9D9D9` 10%) + подпись «BMW S1000RR».
3. **Scene2Select** — сбор сетапа: 4 плитки из `public/setup/` (Куртка, Шлем, Перчатки, Мотоботы), курсор кликает куртку → шлем. Плитки на `cover` + градиент-подпись.
4. **SceneColorScroll** — 3D-изометрический скролл расцветок шлема **AGV K1-S** (фото из `public/colors/k1s_*.jpg`). Курсор `public/pointer.png` (3D Mac pointer) кликает по шлему в перспективе. Угол: `rotateX(40) rotateZ(-24) scale(1.28)`.
5. **Scene3Result** — карточка-результат `generations/20.png` въезжает справа, чипы «✓ Шлем / ✓ Куртка», подпись «Примерь на себя.».
6. **SceneResultsFan** — промежуточная галерея: `20.png` (герой) отъезжает, 17 генераций пружинами разлетаются в кирпичную сетку (3-4-3-4-3), заполняя кадр с обрезкой по краям. Сетка гаснет «вспышкой», морф-карта `1.jpg` плавно проявляется и перетекает в карточку Scene4Deck. `HANDOFF = FAN.indexOf("1.jpg")` **обязан совпадать** с `Scene4Deck CARDS[0]`.
7. **Scene4Deck** — cover-flow: перелистывание 4 генераций из `generations/`. Карточки 760×860, `CARDS[0] = "1.jpg"` (стыкуется с SceneResultsFan). Соседние димятся **оверлеем**, НЕ `filter:brightness`. `f = frame + 3` — срезаны первые 3 статичных кадра.
8. **Scene5Device** — отъезд камеры в телефон `public/device.png` (iPhone 17 Pro Max Deep Blue, 855×1745, прозрачный фон, уже с экраном), бабл «Готово ✓», «Примерочная — прямо в Telegram».
9. **Scene6Outro** — лого **BikeMe** с пробегающим бликом → анимация ребрендинга (TG-шапка «BikeMeBot» → «Ваш магазин» оранжевым wipe) → тэглайн «Примерочная для вашего магазина.» + ценовые строки «SaaS от 399 USD» / «Buy-out 12.000 USD» + контакт. Финальный кадр держится ~1.5с.

Переход — `src/transitions/zoomThrough.tsx` (наплыв из глубины: scale 1.06→1 + blur + кросс-фейд, пружина).
Компоненты — `src/components/MacCursor.tsx` (курсор macOS, inline SVG), `CatalogCard.tsx` (не используется).

## Ассеты (`public/`)
- `source.jpg` — лицо-исходник (совпадает с человеком на примерах → идентичность не «прыгает»).
- `brands.png` — горизонтальная лента лого брендов (8000×833, 10 ячеек).
- `silhouettes/*.png` — силуэты байков (оранжевый на прозрачном; **обрезаны по контенту**).
- `colors/k1s_*.jpg` — фото расцветок шлема AGV K1-S (на белом фоне).
- `setup/jacket.jpg`, `setup/helmet.jpg`, `setup/gloves.jpg`, `setup/boots.jpg` — плитки экипа в Scene2Select.
- `generations/1.jpg … 9.jpg, 10.png … 20.png` — все AI-генерации (результаты). **`20.png` — главный герой** (Scene3Result, Scene5Device, Scene1Intro). `1.jpg` — **handoff-карта** (Scene4Deck CARDS[0], SceneResultsFan HANDOFF).
- `device.png` — iPhone 17 Pro Max Deep Blue, вырезан по альфе (855×1745 RGBA). Уже содержит экран с результатом.
- `pointer.png` — 3D Mac pointer (курсор-палец), используется в SceneColorScroll.
- `db_dump.sql` — дамп БД бота (байки, шлемы, куртки, боты, перчатки, локации, расцветки) — для выбора сетапов под генерации.

## Критичные связки между сценами
- `SceneResultsFan.FAN[HANDOFF]` == `Scene4Deck.CARDS[0]` == `"1.jpg"`. Если меняешь HANDOFF — синхронно меняй CARDS[0] и наоборот.
- `SceneResultsFan.TGT_*` == геометрия карточки Scene4Deck (`CARD_W=760, CARD_H=860, TOP=380` → `TGT_CX=540, TGT_CY=810, TGT_R=36`). При изменении Scene4Deck — обновить TGT_* в SceneResultsFan.
- `Scene3Result` и `Scene5Device` используют `generations/20.png` — одна и та же «главная» генерация.
- `Scene1Intro` использует `source.jpg` — лицо того же человека, что на `20.png`.

## Рабочий процесс (важно)
- **Не рендерить полный MP4 на каждую микроправку** — смотреть в Remotion Studio (скрабинг покадрово).
- Для проверки конкретного кадра — `remotion still --frame=N`.
- **Готчи Remotion:** `filter` на элементе с `overflow:hidden + border-radius` мылит картинку (в Scene4Deck заменено оверлеем); отсутствующий ассет (404) **роняет весь рендер** — всегда сверять имена файлов.
- `Solid` в конце Trailer.tsx — намеренно, не удалять без ведома владельца.
